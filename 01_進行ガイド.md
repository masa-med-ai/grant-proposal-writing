# 01_進行ガイド｜科研費 基盤C・若手研究 調書作成PJT

このファイルは、本PJTの**マスターコントロール**です。
Claudeはこのファイルを起点に作業を進め、各Stepの開始・完了時に必ず `02_状態管理.json` を更新します。
ユーザーは「Step○から再開」「現在の状況を表示」と指示するだけで途中再開できます。

---

## 0. 共通ルール（毎回必ず守ること）

1. **作業前に `02_状態管理.json` を必ず読む**。`current_step` と `completed_steps` を確認してから動く。
2. **作業後に `02_状態管理.json` を必ず更新**する（`last_updated`, `current_step`, `completed_steps`, `notes`）。
3. 中間成果物は **対応するStepフォルダ配下に都度保存**（上書き可）。
4. 重要な決定事項（テーマ、研究期間、マイルストーン等）は `90_成果物/decisions_log.md` に追記。
5. PubMed検索を行ったら **1st author / 雑誌名 / 発刊年 + ハイパーリンク**で引用すること（ユーザー設定）。
6. PubMed検索式は **MeSHで立案 → countで件数検証 → search** の順で行う。
7. 各Stepの完了基準（DoD）を満たしたら、ユーザーに「次のStepに進んでよいか」を必ず確認する。
8. **平易化ポリシー**：想定読者は「博士号レベルだが門外漢の専門家」。一般常識で通じる用語以外は **括弧書きで短い言い換え** を添える。カタカナ語・英略語は段落あたり概ね3語まで、初出時にフルスペル＋日本語訳を明示する。
9. **自己引用ポリシー**：Step1-4 でテーマが確定したら、PubMed MCP で **著者本人（第一著者）の関連原著論文** を検索し、ユーザー合意の上で `13_テーマ候補メモ.md` の「自己引用候補」セクションに記録する。Step3 では「着想に至った経緯」「予備データ」「研究実績」に **1〜3 本** を自然な形で織り込む。

## 3. Step1 詳細手順（Visualizerを使用）

> Step1は **Visualizer（show_widget）を用いてインタラクティブに進める**こと。
> 4つの確認項目を1枚のチェックリストUIにまとめ、ユーザーが状態を一目で把握できるようにする。

### 1-1. Persona / Brand-voice 確認
- 作業ディレクトリ直下に `persona.md`（または `about-me.md`）と `brand-voice.md` があるか確認。
- 無い場合：**ユーザーに「以下のテンプレートで作成してアップロードしてください」と要求**。
  - Persona最低要件：所属、職位、専門領域、これまでの研究実績（代表論文3〜5本）、現在保有しているデータ・検体・コホート、利用可能な設備
  - Brand-voice最低要件：文体（です・ます／である）、得意な表現、避けたい表現
- 結果を `10_Step1_テーマ壁打ち/11_persona_brandvoice_check.md` に保存。

### 1-2. KAKEN MCP 確認
- ツール一覧に `mcp__KAKEN_Search__*` 系（`kaken_search_projects` 等）があるか確認。
- 無ければユーザーに **以下を提示**：
  > KAKENHIデータベース検索MCPが未導入です。下記からBundleをDLしてClaudeに追加することを推奨します。
  > https://github.com/masa-med-ai/KAKENHI-search-mcpbundle
  > ※ Cinii APIキーの取得が必要です（https://api.ci.nii.ac.jp/ja/）。
- 結果を `12_mcp_check.md` に記録。

### 1-3. PubMed MCP 確認
- ツール一覧に `mcp__*pubmed*` 系（`search`, `fetch`, `count` 等）があるか確認。
- 無ければユーザーにインストールを推奨。
- 結果を `12_mcp_check.md` に追記。

### 1-4. テーマ壁打ち
- ユーザーから「ざっくりとした研究テーマ」を1〜2行で入力させる。
- Claudeは以下を意識して具体化を進める：
  - **実現可能性**（Personaのデータ・設備・コホートで実行できるか）
  - **新規性**（KAKEN/PubMedで近年の採択・論文を確認し、被りを避ける）
  - **無謀でないか**（4年や3年で完結する規模か）
- 壁打ちは **3〜5回のラリー**を目安に行い、毎回テーマ候補メモに追記する。
- **必須チェック**：候補テーマごとに以下を明文化してから次へ進む。
  - 学術的「問い」（独立採点項目）
  - 学術的独自性・創造性（独立採点項目）
  - 着想に至った経緯（基盤Cの場合は独立採点項目）
  - 国内外の研究動向と本研究の位置づけ（基盤Cの場合は独立採点項目）
- 最終的に **テーマを1つに絞ってユーザー合意**を取り、`13_テーマ候補メモ.md` の冒頭に「FINAL_THEME」として記載。

### 1-5. 自己引用候補のPubMed探索（FINAL_THEME 確定直後に実施）

> 目的：研究実績・着想経緯・予備データセクションに **さりげなく織り込める著者本人の原著論文** をあらかじめ確保する。

手順：

1. Persona ファイルから著者氏名（英語表記）と所属機関を取得する。
2. PubMed MCP で **第一著者検索式** を立案：
   - 例：`(LastName FM[Author - First]) AND (theme keyword[MeSH] OR ...)`
3. `mcp__pubmed__count` で件数を検証（数十件以内が目安）。多すぎる場合は MeSH を絞る、年代を限定する等で調整。
4. `mcp__pubmed__search` → `mcp__pubmed__fetch` で **タイトル / 雑誌名 / 発刊年 / PMID / 1st author** を取得。
5. 以下の条件で **2〜3本** に絞り込む：
   - FINAL_THEME と直接の親和性がある（先行研究、技術基盤、対象疾患などが重なる）
   - 第一著者である
   - 可能なら **インパクトファクターの高い、または分野で著名な雑誌**
   - できれば **直近5年以内**（古いものは「研究のスタート地点」として扱う場合のみ可）
6. 候補リストをユーザーに提示し、**本文に織り込む論文をユーザーに最終選定**してもらう。
7. 合意した論文を `13_テーマ候補メモ.md` の「## 自己引用候補（Step3 で本文に織り込む）」セクションに **1st author / 雑誌名 / 発刊年 + PubMed リンク** で記載。
8. ユーザーから「この論文も入れたい」と追加提示があれば、そのまま追加して構わない（PubMedで未ヒットの書籍章・症例報告等を含めても良い）。

### Step1 完了条件
- [ ] 11, 12 のチェックがすべて埋まっている
- [ ] 13_テーマ候補メモ.md の先頭に FINAL_THEME が記載され、ユーザーが合意している
- [ ] 13_テーマ候補メモ.md に「自己引用候補」セクションが追記され、最低1本にユーザー合意済み
- [ ] 02_状態管理.json の `current_step` が `Step2` に更新済み

## 4. Step2 詳細手順

### 2-1. 研究期間の設定
- 若手研究：原則2〜4年。基盤C：3〜5年。ユーザーに確認。
- `21_研究期間と全体像.md` に記載（開始年度、終了年度、各年度の概要1行ずつ）。

### 2-2. 研究項目の洗い出し
- FINAL_THEME を達成するための研究項目を **3〜6個程度**に分解。
- `22_研究項目とマイルストーン.md` に Table 形式で記載。

### 2-3. マイルストーン設定（**数値化必須**）
- 各研究項目に対し、**抽象的表現を禁止**。例：
  - NG：「コホートを構築する」
  - OK：「2027年6月までに500例の前向き登録を完了する」
- 各マイルストーンに **開始月／終了月** を付与。

### 2-4. ガントチャートの作成
- `23_ガントチャート.md` に **Mermaid gantt記法**で記述。
- ユーザーに表示して確認 → 必要に応じて修正ラリー。

### Step2 完了条件
- [ ] 21〜23 がすべて埋まり、数値化されたマイルストーンが揃っている
- [ ] ガントチャートにユーザー合意済み

## 5. Step3 詳細手順

### 5-1. ジャンル選択
- 「若手研究」か「基盤C」かをユーザーに確認。
- 該当する記載注意事項（`31_若手研究記載注意事項.md` または `32_基盤C記載注意事項.md`）を**必ず読み込んでから**執筆。

### 5-2. 評定要素の小見出し化
- 評定要素は (1) 内の「・項目」それぞれが**独立採点**される。
- **必ず以下の項目を小見出し（##）として立てる**：
  - 学術的「問い」
  - 学術的独自性・創造性
  - 着想に至った経緯
  - 国内外の研究動向と本研究の位置づけ
  - 研究目的を達成するための研究方法・研究計画
  - 研究目的・研究方法の妥当性・独自性・創造性
- 評定要素が **本文中に埋没しないこと** を最優先。

### 5-3. ドラフト執筆
- `33_研究目的_ドラフト.md` と `34_研究方法_ドラフト.md` を作成。
- Word換算で **4ページに収まる文字量**を目安に試算（1ページ約1,200〜1,400字、合計約5,000〜5,500字を上限の目安）。
- 注：**この段階ではWordファイルを作らない**。ユーザーが最終調整する前提。
- **執筆時、必ず次の2点を意識する**：
  - **平易化（R11/R12）**：想定読者は「博士号レベルだが門外漢の専門家」。専門用語は **括弧書きで短い言い換え** を必ず添える。カタカナ語・英略語は段落あたり概ね3語まで、初出時にフルスペル＋日本語訳。
  - **自己引用の織り込み（R13）**：`13_テーマ候補メモ.md` の「自己引用候補」を**再読**し、「着想に至った経緯」「予備データ」「研究実績」のいずれかに **1〜3 本** を自然に埋め込む。自慢調や羅列にならないよう、先行研究として淡々と参照する。

### 5-4. 仕上げチェック（ドラフト完了後に必ず実施）

| チェック項目 | 内容 |
|------------|------|
| 平易化チェック | 1段落ずつ読み返し、専門用語に短い言い換えが付いているか／カタカナ語が密集していないかを確認 |
| 略語チェック | 初出のすべての略語に「フルスペル＋日本語訳」が併記されているか |
| 自己引用チェック | `13_テーマ候補メモ.md` の自己引用候補から最低1本が本文に織り込まれているか |
| 引用フォーマット | 1st author / 雑誌名 / 発刊年 + ハイパーリンクの統一 |
| 評定要素の独立性 | A〜F（基盤Cは加えてG1/G2）それぞれに対応する `##` 小見出しが立っているか |

### Step3 完了条件
- [ ] 該当する記載注意事項を読み込んだ
- [ ] 33, 34 に評定要素ごとの小見出しが揃っている
- [ ] 文字数試算が4ページ以内に収まっている
- [ ] 5-4 の仕上げチェック（平易化・自己引用・引用フォーマット）を通過済み
- [ ] 02_状態管理.json を Step4 に更新

## 7. 中断・再開時のClaudeの初動

ユーザーが「再開して」「現在の状況は？」と尋ねたら、Claudeは以下を実行：

1. `02_状態管理.json` を Read
2. `current_step` と直近の `notes` をユーザーに要約提示
3. 「Step○の続きから進めます。よろしいですか？」と確認
4. 合意後、当該Stepの手順に沿って作業を再開
